# 语言层：AI 痕迹库

合并自四个来源，按中文技术写作场景去重提纯：维基百科《Signs of AI writing》（经 blader/humanizer、op7418/Humanizer-zh 汉化）、OUBIGFA/De-AI 的中文原生痕迹库、ai-zixun/humanizer-zh 的翻译腔清单，以及从本 skill 作者旧稿中校准出的个人残留模式。

**先读这条再用清单**：黑名单不是本质。范文《giffgaff 使用指南》大量使用粗体标题列表、数字步骤、"一句话总结"——全是清单上的"痕迹"，但每一处都承载着挣来的信息，所以读起来是人话。判断任何模式去留的标准只有一个：**删掉这个形式，信息还在吗？形式在帮读者扫读，还是在掩饰内容空洞？** 机械执行黑名单会把文章磨成没有声音的说明书，那是另一种 AI 味。

修补顺序：结构痕迹 → 语气痕迹 → 句式痕迹 → 词汇痕迹 → 排版痕迹。词语替换放最后，因为上游手术常会顺手消灭下游问题。

## 一、结构痕迹

| 痕迹 | 识别 | 修法 |
|------|------|------|
| 段落同构 | 连续 3 段都是"观点句+解释+段尾小结" | 改变信息进入方式：用事实、场景、引语开段；段尾敢停在具体后果上不总结 |
| 知识卡片化 | 引用块/callout 密度过高，正文被切成均匀的知识点卡片 | 卡片占比 ≤ 1/3；把"XX 是什么"类卡片熔回正文叙述，只留真正的警告和旁注 |
| 提纲扩写节 | 小节形态雷同：引入一句+图+问答块 | 结构层手术解决（见 structure.md），语言层治不了 |
| 挑战与展望节 | "尽管存在这些挑战……未来可期" | 整节删掉，换成具体的下一步或已知的未解问题 |
| 通用积极结尾 | "让我们拭目以待""前景广阔" | 停在判断、行动或后果上 |
| 均匀段落 | 每段长度、句数几乎一样 | 允许一句成段做强调，也允许厚段做论证，按呼吸排 |

## 二、语气痕迹

| 痕迹 | 识别 | 修法 |
|------|------|------|
| 协作口吻 | "希望这对您有帮助""作为AI""如果您想让我继续" | 直接删，正文不是聊天记录。注意区分："接下来我们逐层分析""下面来看"是作者正常的行文推进语，保留 |
| 报账式开头结尾 | "收到""本文将介绍""以上就是……的全部内容" | 开头直接进 SCQA，结尾直接停 |
| 讲义动作 | "拆一拆""盘一盘""划重点""敲黑板"，无必要的"说白了""本质上""归根结底" | 删动作词，直接说事 |
| 设问自答 | "XX 是什么？答案是……""为什么 Y？因为……"，包括设问式小标题 | 全文设问自答小标题 ≤ 2；改为陈述句标题或读者利益标题 |
| 读者许诺 | "读完你就能""看懂这个例子就懂了" | 删许诺，内容自己会证明 |
| 教学式二人称 | 满篇"你会发现""你需要" | "你"只用于少量归谬或设想；技术文默认第一人称视角 |
| 谄媚语气 | "好问题！""您说得完全正确" | 删 |
| 知识截止免责 | "截至目前""根据可用信息" | 有确切信息给信息，没有就说自己没查到 |

## 三、句式痕迹

| 痕迹 | 识别 | 修法 |
|------|------|------|
| 二分对照壳 | "不是 A 而是 B""不在于 A 而在于 B""这不仅仅是……更是……" | 全文 ≤ 1；多数场景直接说 B 就够了 |
| 三项排比 | "无缝、直观且强大"，什么都凑三个 | ≤ 1；两项或四项都比三项自然，或者干脆只说最重要的那个 |
| 条件句堆叠 | "一旦……就……""只有……才……""无论……都……""通过……来……" | 合计 ≤ 2；多数可改成直陈 |
| -ing 式肤浅分析 | 句尾挂"，体现了/彰显了/凸显了/为……奠定了基础" | 整个尾巴删掉；真有意义就单独成句写透 |
| 虚假范围 | "从底层架构到用户体验"（两端不在一个尺度上） | ≤ 1；改成实指列举 |
| 系动词回避 | "作为/充当/扮演着……的角色""拥有/配备了" | 直接用"是""有" |
| 同义词轮换 | 主人公/主要角色/中心人物/英雄轮着换 | 同一事物用同一个名字，重复不是罪 |
| 过度限定 | "可能在某种程度上或许会" | 一个限定词就够；有把握就不限定 |
| 金句欲 | 一句话明显在等着被截图转发 | 重写成朴素版；判断的力度靠证据不靠句式 |

## 四、词汇痕迹

**高频 AI 词（堆叠即嫌疑，出现两个以上必查）**：赋能、抓手、闭环、底层逻辑、深耕、洞察、践行、驱动、构建、打造、拆解、梳理、剖析、解构、聚焦、格局、图景、织锦、里程碑、新篇章、彰显、凸显、擘画。

**空大词（无机制解释不得使用）**：颠覆、革命、重塑、深刻改变、质的飞跃、划时代。

**万能形容词（一律替换为具体事实或删除）**：全面、强大、无缝、至关重要、极具价值、令人惊叹、优雅（形容代码时慎用）。

**装饰性口语（硬装生动，删）**：补一刀、一锤定音、王炸、天花板、杀疯了。

**强调性拐杖词（合计 ≤ 2）**：更关键的是、更要命的是、值得注意的是、总之、与此同时、首先/其次/最后。注意和自然逻辑连接词区分开：因此、所以、其实、实际上、也就是说、换个角度说，这些是人说话的正常黏合剂，照常使用，不许为了"干净"全部剥掉——剥光了就是 humanizer 腔（见第七节）。

**戏剧化揭露（删）**：遮羞布、面具、外衣、揭开真面目、戳穿真相。

**翻译腔连接（改写为中文语序）**：对于……来说、基于……、围绕……展开、使得……得以、在……的加持下、就……而言。英文句法直译的长定语句拆成短句。

## 五、排版痕迹

| 痕迹 | 修法 |
|------|------|
| 粗体滥用 | 粗体只标关键术语首次出现和真正的警告；一段超过两处粗体必删 |
| 粗体标题式列表（**标题：**解释） | 信息真实且并列时可保留，但检查是否该熔成叙述；空洞时整个列表删 |
| 数字清单开路（"做了四件事""6 个要点"） | 全文 ≤ 2；清单项之间没有并列关系时改叙述 |
| emoji 装饰 | 技术文正文禁用 |
| 修辞性破折号成对使用 | 改逗号、冒号或拆句；破折号全文 ≤ 2 处 |
| 表格滥用 | 表格只装可枚举的短事实；解释性内容回到正文 |
| 弯引号/引号混乱 | 中文用「」或""，统一全篇 |
| 中英文假空格 | "Agent 引擎""160 行"这类盘古式空格是一眼可辨的AI排版痕迹，全文去掉：中文与英文、中文与数字之间不留空格（写"Agent引擎""160行"）；NBSP、全角空格、零宽字符等特殊空白同样清除；代码块内的引用代码不改 |
| 标题层级过深 | 博客文 ≥ 4 级标题说明结构出了问题，回结构层 |

## 六、个人残留模式（作者校准，优先检查）

从作者旧稿《Agent 引擎实现原理》诊断出的高频残留，处理作者的稿件时最先扫这几条：

1. **callout/引用块密度过高**：40+ 个卡片把博客切成了参考文档。发布版正文优先，卡片 ≤ 1/3。
2. **设问自答式小标题成灾**："为什么要有状态机""CountDownLatch 是什么""数字员工是干什么的"连排。保留最有价值的 1-2 个，其余熔进叙述。
3. **"做 N 件事"清单开路**："think 做四件事""act 做四件事""值得学的 5 个设计""6 个局限"。数字清单是压缩的提纲，不是消化过的叙述。
4. **小节模板同构**：引入一句 + 图 + 问答卡片，十四节一个模子。
5. **全文无"我"**：没有一处真实认知轨迹。按 argumentation.md 的"个人在场"补，素材必须来自作者本人。
6. **穷尽式覆盖**：每个类每个方法都要讲。结构层手术解决。

## 七、矫枉过正：humanizer 腔（同样是 AI 痕迹，同样要修）

机械执行前六节的清单，会产出第二种 AI 腔：句句压缩、字字铿锵、冷得像电报。文章要有温度，温度来自完整的句子、自然的语气词和连接词、以及一个真的在跟你说话的人。特征和修法：

| 痕迹 | 识别 | 修法 |
|------|------|------|
| 电报句 | 判断句被压到最短："光靠聊天不够""管的就是这个循环" | 写完整："光靠聊天肯定是不够的"；"是……的""实际上""其实"该有就有 |
| 金句连发 | 段段以铿锵短句收尾，通篇像语录合集 | 收尾说平实的话；一篇文章留一两处力道就够了 |
| 连接词剥光 | 因此/所以/也就是说全被删掉，句间逻辑靠读者脑补 | 自然连接词是说话的黏合剂，照常用 |
| 裸场景开头 | "让 LLM 做一份报告"直接砸到读者脸上 | 用"假如/比如/想象一下"把人带进场景 |
| 过度压缩用词 | 满篇"得/干/管/砸/怼"，名词也硬凹短词："解法""玩法""打法" | 工程师的正式书面语和口语自然混用："实现方案""技术架构""逐层分析"该用就用，全是压缩词也是一种腔 |
| 零温度 | 通篇正确但冷冰冰，没有一句像人当面说的话 | 允许感叹、允许自嘲、允许"说实话这块我也琢磨了挺久"；温度不是加语气词，是作者真的在场 |

自检方法：随机挑三句念出来，问自己当面给同事讲解时会不会这么说。笔下写的是"Agent 引擎管的就是这个循环"，嘴上说的是"所以 Agent 引擎实际上就是这样一套机制"——以嘴上那版为准。
